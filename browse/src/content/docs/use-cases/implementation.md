Реализация — это место, где большинство инженеров впервые сталкиваются с агентами. Успех зависит от подхода, а не только от промптинга.

## Высокоценные задачи

**Генерация шаблонного кода** — CRUD-операции, API-каркас, компоненты форм, DTO, конфигурации. Агенты справляются с этим быстро и надёжно. Именно здесь заявления о "10x" почти верны.

**Реализация функциональности** — Лучше всего работает, когда требования ясны, паттерны в вашем кодовой базе существуют для подражания, а область ограничена (один PR, несколько файлов).

**Исправление багов** — Ясные баги — отличные задачи для агентов. "Вход пользователя не работает с null pointer, когда email содержит '+'" лучше, чем "почините поток входа."

**Рефакторинг** — Механический рефакторинг идеален: переименование по кодовой базе, извлечение функции/класса, конвертация паттернов. Сложное переструктурирование даётся труднее.

**Трансформации данных** — Скрипты миграции, конвертации форматов, ETL-логика. Чётко определённые входы и выходы делают это дружественным к агентам.

## Рабочий процесс

1. **Планируйте** — Знайте, какие файлы изменятся, желаемое конечное состояние и что НЕ должно измениться.

2. **Устанавливайте контекст** — Предоставляйте релевантный код, существующие паттерны для подражания и ограничения.

3. **Генерируйте** — Для сложных задач сначала запросите план: "Опишите, как бы вы реализовали [функцию]. Пока не пишите код."

4. **Валидируйте** — Это работает? Обрабатывает краевые случаи? Следует соглашениям? Есть проблемы безопасности?

5. **Уточняйте** — "Здесь не обрабатывается случай, когда..." или "Используйте паттерн из [файла] вместо этого."

## Что замедляет

- **Слишком сложные промпты** — Начните просто, добавляйте детали по необходимости
- **Недостаточно ограниченные запросы** — "Построй функцию" оставляет слишком много решений агенту
- **Противостояние агенту** — После 3+ повторных промптов без прогресса переосмыслите или сделайте сами
- **Недостаточный контекст** — Отсутствие контекста приводит к неверному результату
- **Неправильный инструмент** — Некоторый код быстрее написать вручную

## Паттерны промптов

**Реализация функциональности:**

```
Реализуйте [функцию] в [файл/модуль].

Требования:
- [Конкретное требование 1]
- [Конкретное требование 2]

Следуйте паттерну из [существующего примера].
Не изменяйте [что сохранить].
```

**Исправление бага:**

```
Баг: [описание]
Воспроизведение: [шаги или код]
Ожидаемое: [поведение]
Фактическое: [поведение]

Исправьте это в [файл]. Корневая причина — [если известна].
```

**Рефакторинг:**

```
Проведите рефакторинг [модуль/функция] к [желаемому изменению].

Сохраните публичный API неизменным.
Поддерживайте всю существующую функциональность.
[Дополнительные ограничения]
```
