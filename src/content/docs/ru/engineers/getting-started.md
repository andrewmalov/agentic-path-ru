Выберите один инструмент и изучите его хорошо, прежде чем попытаться всё использовать.

## Выберите свой первый инструмент

**Новичок в AI-программировании?** Начните с GitHub Copilot или аналогичного. Низкий риск, мгновенная ценность.

**Готовы к большей автономии?** Попробуйте задачный агент, такой как Cursor, Cline или Kilo Code, для изменений в нескольких файлах.

**Исследуете?** Большинство инструментов имеют бесплатные уровни. Попробуйте несколько, придерживайтесь одного для глубокого изучения.

## Ваша первая сессия

Начинайте с малого — не генерируйте весь свой проект.

**Хорошие первые задачи:**

- Сгенерировать одну функцию из чёткого описания
- Написать тесты для существующего кода
- Добавить документацию к запутанному коду
- Рефакторинг небольшого, мessy кусочка

**Наблюдайте, как это работает:** Обратите внимание, какой контекст используется, как обрабатывается неоднозначность, где делаются ошибки.

## Развивайте интуицию для подсказок

**Будьте явными:** Вместо "исправь эту ошибку" попробуйте "Функция `calculateTotal` возвращает NaN, когда массив items пуст. Добавь проверку, которая возвращает 0."

**Предоставляйте контекст:** "Это React компонент, использующий TypeScript. Следуйте шаблону в других компонентах в этой папке."

**Устанавливайте ограничения:** "Не изменяй публичный API. Сохраняй обратную совместимость."

## Когда делегировать

Не каждая задача должна отправляться агенту. Спросите себя:

1. **Насколько чётка задача?** Неопределённые задачи падают. Чёткие задачи успевают.
2. **Сколько контекста нужно?** Глубокие предметные знания рискованны для делегирования.
3. **Каков радиус последствий?** Ошибки в критических участках стоят дороже для исправления.
4. **Сколько времени это займёт?** Если это 5 минут вручную, то подсказка может не оказаться оправданием.

### Хорошие кандидаты

| Тип задачи                           | Почему это работает                                        |
| ------------------------------------ | -------------------------------------------------- |
| **Боилерплейт** (CRUD, DTOs, config) | Повторяющееся, чётко определённое, низкорисковое                     |
| **Тесты**                            | Самовалидирующиеся — вы сразу знаете, работают ли они  |
| **Документация**                      | Легко проверить точность                            |
| **Механический рефакторинг**           | Переименовывание, извлечение функций, преобразования синтаксиса |
| **Исправление багов с чётким воспроизведением**       | "Когда X происходит, Y происходит, но должно быть Z"        |

### Оставьте для себя

- **Архитектурные решения** — Агенты не понимают историю вашей системы или её будущее
- **Безопасность** — Стоимость тонких ошибок слишком высока
- **Производительность** — Агенты оптимизируют на корректность, а не на скорость
- **Новые алгоритмы** — Агенты находят паттерны; новые задачи нуждаются в человеческом творчестве
- **Неопределённые требования** — Уточняйте перед делегированием

### Серая зона

Для задач, которые не подходят чётко: **начните с агента, будьте готовы взять управление.** Получите начальную структуру от агента, затем доработайте вручную.

## Стройте привычки

**Неделя 1:** Боилерплейт и тесты только  
**Неделя 2:** Добавьте документацию и рефакторинг  
**Неделя 3:** Реализация функций с чёткими спецификациями  
**Неделя 4:** Сложные, многошаговые задачи

## Знайте, когда остановиться

Признаки, когда вам стоит кодить самому:

- Вы переподсказывали 3+ раз без прогресса
- Задача требует глубокого контекста, которого у агента нет
- Вы могли закончить вручную за время, потраченное на подсказки

Нет ничего позорного в ручном кодинге. Цель — продуктивность, а не использование агентов.
